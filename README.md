## 是什么

**Ray-CEO-SKILLS** 是一个为 CEO 角色设计的 skill 仓库。

它重点解决这几类问题:

1. **战略知识分散**: 想法很多,但没有结构化沉淀,每次都要重新组织。
2. **表达成本高**: 做演讲、写方法论、讲产品故事时,很容易从 0 开始。
3. **团队标准不统一**: 同样的任务,不同 agent / 不同成员产出风格不稳定。
4. **AI 能力难复用**: 做过一次的好工作流,没有变成可以复用的 skill。
5. **知识和执行脱节**: 有框架,但没有真正变成可操作的 SOP、模板和工具链。

这个仓库希望沉淀的不是“资料”,而是 **可复用能力**:

- CEO 的思考框架
- 对外表达与讲故事能力
- 组织知识管理能力
- 演讲与 deck 生产能力
- 学习、辅导、训练型能力

> **一句话总结**: 把 CEO 常用的高价值能力,做成可以长期复用、持续扩展、能被 AI 执行的 skill collection。

当前仓库已经收录的核心 skill:

| Skill | 作用 | 适合场景 |

| **OpenClaw-Enterprise-Knowledge-Base-Skill** | 企业知识库与结构化沉淀 | 内部 wiki、知识运营、信息归档 |

未来适合继续加入的方向:

## 这个仓库和普通知识库的区别

| 维度 | 普通笔记仓库 | Prompt 收藏夹 | **Ray-CEO-SKILLS** |
|---|---|---|---|
| 组织方式 | 文档为主 | prompt 为主 | **skill + workflow + playbook** |
| 可执行性 | 低 | 中 | **高** |
| 复用性 | 靠人工翻找 | 靠记忆 | **按目录和 skill 复用** |
| 对 CEO 的帮助 | 记录 | 灵感 | **决策、表达、沉淀、执行** |
| 与 AI 协作 | 弱 | 中 | **强** |

## ❓ FAQ

**这是一个单独 skill 吗?**
不是。它更像一个 **CEO skills collection / registry**，里面可以包含多个 skill。

**为什么里面既有知识库 skill，也有学习辅导 skill?**
因为这个仓库目前是在收“高复用 AI 能力模块”，而不是只限定单一行业。后续可以继续按 CEO 视角重组分类。

**之后应该继续往哪里扩展?**
最值得补的是 `strategy/`、`leadership/`、`operations/` 和 `playbooks/`。

**这个仓库适合公开吗?**
适合公开“通用方法论”和通用 skill，不适合直接放敏感经营信息或私有内部数据。

**下一步最值得做什么?**
给每个 skill 补清楚“输入是什么 / 输出是什么 / 什么时候用 / 一个最佳示例”。
